# 🎯 Exam: AI Programming

> Format: Live coding with AI
>
>
> **Duration**: 20-60 minutes per person
>
> **Grading**: AI agents + instructors
>
> **Result**: 0-100 points
>
> **Can be taken at any point during the semester**

---

## ⚡ Exam Format

### What Happens in 60 Minutes

**Structure:**

1. **0-2 minutes**: You receive an unknown task
2. **2-18 minutes**: You build the first MVP
3. **8-10 minutes**: You document the solution
4. **10-30 minutes**: You evolve the MVP into a more advanced version

**What you need to demonstrate:**

- Building a working solution from scratch
- Ability to iterate quickly
- Evolution from simple to complex
- Effective use of AI

### Example Tasks and Evolutions

- Telegram bot (MVP) → adding a database → analytics → tests
- TODO list → API → synchronization → multi-user
- Simple parser → error handling → caching → scaling

## 📊 Grading System

### Point Distribution

| Criterion | Points | What is measured | Who grades |
| --- | --- | --- | --- |
| **Time to MVP** | 25 | Time to first working version | AI agent |
| **Token efficiency** | 25 | Your tokens vs group median | AI agent |
| **Quality of evolution** | 25 | How much the functionality has grown in complexity | AI + human |
| **Tests** | 15 | Whether they exist and pass | AI agent |
| **Documentation** | 10 | Clarity of the solution | AI agent |

### Competitive Element

Your metrics are compared against other students working on the same task:

- **Top 25%** → maximum points
- **Median** → average points
- **Bottom 25%** → minimum points

This means your grade partially depends on how effectively you use AI compared to your classmates.

---

## 🧪 Testing Requirements (15 points)

### Minimum Requirements

- At least 2-3 tests that actually verify something meaningful
- Any type: unit, integration, or e2e
- AI can write them — the important thing is that they work
- Tests must be capable of failing (not always green)

### What Earns Bonuses

- Test-first approach (+3 points)
- Coverage of main use cases (+3 points)
- Tests for new features during evolution (+3 points)

**Important**: This is not a TDD exam. Tests are simply an indicator that you understand code should work correctly.

## 📈 Criteria for Good Evolution

### MVP1 → MVP2

**What counts as evolution:**

- Adding 2-3 meaningful features
- Improving architecture
- Adding persistence
- Integrating with external services
- Improving UX/UI
- Handling errors and edge cases

**What does NOT count as evolution:**

- Cosmetic changes
- Refactoring without new functionality
- Adding comments

---

## 🤖 How AI Grading Works

### What Agents Check Automatically

1. **Functionality**: Does the code run? Do core features work?
2. **Task compliance**: Has the given problem been solved?
3. **Efficiency metrics**: Tokens, time, number of iterations
4. **Tests**: Do they run? Do they pass? Can they fail?
5. **Evolution**: How much has the functionality grown in complexity?

### Grading Transparency

After the exam you will receive:

- A detailed report for each criterion
- A comparison of your metrics against the group
- A recording of your session
- The option to appeal your grade within 48 hours

## 📝 Frequently Asked Questions

**Q: What if I have a slow internet connection?**
A: This is a critical issue. You need a stable connection to work with AI.

**Q: How difficult will the tasks be?**
A: Mid-level developer difficulty. The emphasis is on speed, not algorithms.

**Q: Can I use multiple AI tools at the same time?**
A: Yes, and it is even encouraged.

**Q: What if I don't have time to complete the evolution?**
A: A solid MVP is better than an unfinished evolution.

**Q: Is code quality and style taken into account?**
A: No, only whether it works.

**Q: Can I prepare templates in advance?**
A: Yes, but the main code must be written during the exam.

**Q: What if the AI returns an error or refuses to help?**
A: Have a backup tool ready. This is part of the skill of working with AI.

## 💭 Exam Philosophy

We are not testing your knowledge of programming languages or design patterns. We are evaluating your ability to rapidly build working solutions in symbiosis with AI.

The exam closely mirrors real-world work in 2025: an unknown task, limited time, and any tools at your disposal.

If you practiced regularly throughout the course, the exam will be just another project. If not, it will be tough.
