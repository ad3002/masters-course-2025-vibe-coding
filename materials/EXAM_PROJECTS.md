## Telegram-боты

### Проект 1: Birthday Reminder Bot
**Стек**: aiogram 3.x, SQLite, pytest-bdd
**Функционал**: Напоминания о днях рождения, CRUD операции, форматирование дат
**BDD**: 4 .feature файла

### Проект 2: Gym CLI + Telegram Bot
**Стек**: Click CLI, aiogram, Whisper API, GPT-4o-mini
**Функционал**: Дневник тренировок с голосовым вводом, понимает разговорные формулировки ("добавь жим лёжа восемьдесят кило")
**Особенность**: LLM парсер голоса

### Проект 3: Run Stats CLI
**Стек**: Click CLI, SQLite, pytest-bdd
**Функционал**: Дневник пробежек (add/week/month/best/progress/compare/delete), поддержка км/миль/метров
**BDD**: 8 .feature файлов на русском (352 строки)

### Проект 4: Pizza Telegram Bot
**Стек**: aiogram 3.x, SQLite, pytest-bdd
**Функционал**: Учёт долгов в группе, автоматический netting, изоляция чатов
**BDD**: 9 .feature файлов (285 строк)
**Архитектура**: Repository pattern

### Проект 5: NLExam — Учёт расходов
**Стек**: YaGPT, Yandex SpeechKit, YDB, FastAPI, Yandex Serverless
**Функционал**: Голосовой ввод расходов, YaGPT парсинг ("тыщу на такси"), аналитика
**BDD**: 12 .feature файлов (334 строки)
**Особенность**: Production deployed, multi-agent pipeline (5 агентов)

### Проект 6: Debt Bot + Web App
**Стек**: aiogram, Flask, REST API, Docker
**Функционал**: Telegram бот + веб-интерфейс для учёта долгов
**BDD**: 10 .feature файлов (297 строк)
**Особенность**: Full-stack (Bot + Web + Docker), 6 ролей разработки

### Проект 7: Meeting Room Booking Bot
**Стек**: aiogram, SQLite, pytest-bdd, Docker
**Функционал**: Бронирование переговорок, 13 команд (7 user + 6 admin), timezone support
**BDD**: 5 .feature файлов
**Архитектура**: Clean Architecture (Repository + Service Layer)

### Проект 8: Movie Selection Bot (пары/группы)
**Стек**: aiogram, SQLite, pytest
**Функционал**: Выбор фильмов для пар/групп, списки желаний, история просмотров
**Особенность**: 25 BDD сценариев в SPEC.md, ручное тестирование с реальными пользователями

### Проект 9: MovieBot
**Стек**: aiogram 3.x, SQLite, pytest-bdd
**Функционал**: Списки желаний, пересечение списков, умный выбор (не повторяет, не предлагает просмотренные)
**BDD**: 6 .feature файлов на русском (262 строки)
**Архитектура**: Clean Architecture (handlers → services → repositories)

## Паттерны успешных проектов

### Структура BDD на русском
```gherkin
# language: ru

Функциональность: Добавление фильмов в список желаний
  Как пользователь бота
  Я хочу добавлять фильмы в свой список
  Чтобы потом выбрать что посмотреть

  Сценарий: Успешное добавление фильма
    Допустим пользователь зарегистрирован
    Когда он отправляет "хочу посмотреть Оппенгеймер"
    Тогда бот отвечает "Добавил в твой список"
```

### Типичная архитектура
```
project/
├── src/
│   ├── bot/
│   │   └── handlers/      # Тонкий слой — только парсинг и ответы
│   ├── services/          # Бизнес-логика (без Telegram)
│   └── database/
│       └── repositories/  # Работа с БД
├── tests/
│   ├── features/          # .feature файлы (BDD)
│   └── steps/             # Step definitions
├── CLAUDE.md              # Инструкции для AI
└── README.md
```

### Рекомендуемый workflow
```
1. Создать .feature файлы (спецификация)
2. Написать step definitions (связка с кодом)
3. Реализовать код (чтобы тесты прошли)
4. Коммитить часто (15-50 коммитов за экзамен)
5. Документировать в CLAUDE.md
```

## Что отличает лучшие работы

1. **BDD-first** — тесты написаны ДО кода (видно по истории коммитов)
2. **Русские Gherkin сценарии** — `# language: ru`
3. **Clean Architecture** — разделение handlers/services/repositories
4. **Частые коммиты** — атомарные изменения с понятными сообщениями
5. **CLAUDE.md** — документация workflow с AI
6. **Реальное тестирование** — некоторые тестировали с друзьями/семьёй
